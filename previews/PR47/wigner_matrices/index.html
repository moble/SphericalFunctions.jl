Wigner's $ùîá$ and $d$ matrices

Wigner's $ùîá$ matrices ‚Äî and to a lesser extent, the related $d$ matrices ‚Äî are extremely important in the theory of rotations. Each element is, itself, a special function of the rotation group: in particular, an eigenfunction of the left- and right-Lie derivatives, and thus a spin-weighted spherical function. Collectively, they describe how spin-weighted spherical functions transform under rotation. But their accurate and efficient computation is surprisingly subtle. This package implements the current state-of-the-art techniques for their fast and accurate computation, based on the $H$ recursion.

The actual computations can be done with the D_matrices function:

using Quaternionic
using SphericalFunctions

R = randn(RotorF64)
‚Ñì‚Çò‚Çê‚Çì = 8
ùîá = D_matrices(R, ‚Ñì‚Çò‚Çê‚Çì)

However, the matrices can take up a lot of memory. So for maximum efficiency when calling this function repeatedly with different R values, it is best to pre-allocate the necessary memory with the D_prep function, and the pass that in as an argument to D_matrices!:

D_storage = D_prep(‚Ñì‚Çò‚Çê‚Çì)
ùîá = D_matrices!(D_storage, R)

(Beware that, as noted in the documentation for D_matrices!, the output ùîá is just a reference to part of the D_storage object, so you should not reuse D_storage until you have copied or otherwise finished using ùîá.)

The output ùîá is a (linear!) vector of Complex numbers with the same base type as R. The ordering of the elements is described in the documentation for D_matrices. It is also possible to efficiently view slices of this vector as a series of individual matrices using a D_iterator:

for (‚Ñì, DÀ°) in zip(0:‚Ñì‚Çò‚Çê‚Çì, D_iterator(ùîá, ‚Ñì‚Çò‚Çê‚Çì))
    # Do something with the matrix DÀ°[‚Ñì+m‚Ä≤+1, ‚Ñì+m+1]
end

For the $d$ matrices, we have almost the same interface, except that instead of the input quaternion R we only need the angle Œ≤ (or its complex angle expiŒ≤, which can be computed directly in some cases), and the output is real-valued:

using Quaternionic
using SphericalFunctions

Œ≤ = œÄ * rand(Float64)
‚Ñì‚Çò‚Çê‚Çì = 8
d = d_matrices(Œ≤, ‚Ñì‚Çò‚Çê‚Çì)

Again, for repeated calls, it is best to pre-allocate storage:

d_storage = d_prep(‚Ñì‚Çò‚Çê‚Çì)
d = d_matrices!(d_storage, Œ≤)

The output d is a vector of numbers of the same type as Œ≤, ordered in the same way as the output of D_matrices. And similarly, we can iterate over the individual matrices using a d_iterator.

Compute Wigner's ùîá matrices $\mathfrak{D}^{(\ell)}_{m',m}(\beta)$ for all $\ell \leq \ell_\mathrm{max}$.

In all cases, the result is returned in a 1-dimensional array ordered as

[
    ùîáÀ°‚Çò‚Çö,‚Çò(R)
    for ‚Ñì ‚àà 0:‚Ñì‚Çò‚Çê‚Çì
    for mp ‚àà -‚Ñì:‚Ñì
    for m ‚àà -‚Ñì:‚Ñì
]

When the first argument is D, it will be modified, so it must be at least as large as that array. When the first argument is D_storage, it should be the quantity returned by D_prep, and the result will be written into the D field of that tuple. Both of these options ‚Äî especially the latter ‚Äî reduce the number of allocations needed on each call to the corresponding functions, which should increase the speed significantly. Note that the D or D_storage arguments must have types compatible with the type of R or Œ±, Œ≤, Œ≥.

Warn

When using the D_storage argument (which is recommended), the returned quantity D will be an alias of D_storage[1]. If you want to retain that data after the next call to D_matrices!, you should copy it with copy(D).

The Œ±, Œ≤, Œ≥ arguments are Euler angles as described in the documentation of Quaternionic.from_euler_angles.

See also D_matrices for a simpler function call when you only need to evaluate the matrices for a single value of R or Œ±, Œ≤, Œ≥.

This iterator mistakenly returns the transpose of the result implied by the documentation. To avoid breaking code that relies on this behavior, this is a final release in this major version of the package.
